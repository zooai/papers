\begin{abstract}
We present \cevm{}, a C++ GPU-accelerated Ethereum Virtual Machine achieving \textbf{2.3 billion gas operations per second} (Gops/sec) on a single data-center GPU.
\cevm{} partitions each block into a maximal independent set executed in parallel on GPU streaming multiprocessors and a residual conflict set executed sequentially.
Post-execution conflict detection triggers deterministic re-execution, guaranteeing identical state roots across all validators regardless of hardware or scheduling order.
On Zoo Network's L2 workloads, empirical conflict rates average 4.7\%, yielding $23.8\times$ speedup over single-threaded CPU EVM execution.
Multi-GPU configurations scale linearly, sustaining 8.9~Gops/sec on four GPUs.
All results are mechanically verified in Lean~4 (\texttt{GPU/EVMScaling.lean}), with proofs of linear scaling, Amdahl's bound, and deterministic re-execution correctness.
\end{abstract}

\section{Introduction}
\label{sec:intro}

The EVM was designed for single-threaded deterministic execution.
Every transaction executes sequentially against the prior transaction's state, producing a canonical state root.
This guarantees determinism but imposes a hard throughput ceiling: a single CPU core processes roughly 100~Mops/sec, bounding L1 Ethereum to 15--30 TPS \cite{wood2014ethereum}.
L2 rollups inherit this bottleneck---execution remains the critical path regardless of data-layer capacity.

Zoo Network operates as a specialized L2 on \Lux{}, serving AI-native workloads: decentralized inference (AI Mining), privacy-preserving computation (FHE VM), and on-chain DeFi (DEX VM) \cite{zoo-network-architecture}.
A single AI Mining round produces 2,000--10,000 result-submission transactions within a 12-second block.
DEX arbitrage bursts exceed 5,000 TPS.
FHE ciphertext operations consume 10--100$\times$ the gas of standard transfers.

\cevm{} addresses this gap by moving EVM execution to GPU hardware.
The key insight: \emph{most transactions are independent}---they touch disjoint storage slots.
Only 3--8\% on Zoo L2 exhibit true state conflicts.
By isolating conflicts, \cevm{} executes the independent majority in parallel while preserving full determinism.

\subsection{Contributions}

\begin{enumerate}[leftmargin=*]
\item \textbf{Transaction dependency analysis} partitioning blocks into independent and conflict sets in $O(n \log n)$.
\item \textbf{GPU execution engine} in C++/CUDA achieving 2.3~Gops/sec on a single NVIDIA H100.
\item \textbf{Deterministic conflict resolution} ensuring identical state roots across heterogeneous hardware.
\item \textbf{Mechanically verified proofs} in Lean~4 of linear scaling, Amdahl's bound, and re-execution correctness.
\item \textbf{Integration architecture} for Zoo L2 with DEX VM, FHE VM, and AI Mining precompiles.
\end{enumerate}

\section{Architecture}
\label{sec:architecture}

\cevm{} comprises four stages per block: dependency analysis, parallel execution, conflict detection, and sequential re-execution.

\subsection{Transaction Dependency Analysis}

\begin{definition}[Read-Write Set]
For transaction $t$, let $R(t) \subseteq \mathcal{S}$ denote state slots read and $W(t) \subseteq \mathcal{S}$ slots written, where $\mathcal{S}$ is the global state-slot universe.
\end{definition}

\begin{definition}[Independence]
Transactions $t_i, t_j$ are \emph{independent} iff:
\begin{equation}
W(t_i) \cap W(t_j) = \emptyset \;\wedge\; W(t_i) \cap R(t_j) = \emptyset \;\wedge\; R(t_i) \cap W(t_j) = \emptyset
\end{equation}
\end{definition}

\cevm{} estimates read-write sets via static analysis of calldata and contract bytecode.
For simple transfers and well-typed calls, analysis is exact.
For dynamic dispatch (\texttt{DELEGATECALL} chains), it over-approximates, conservatively placing ambiguous transactions in the conflict set.

The dependency graph $G = (T, E)$ has edge $(t_i, t_j) \in E$ iff $t_i, t_j$ are not independent.
The maximal independent set $I \subseteq T$ is computed via greedy graph coloring in $O(n \log n)$, with conflict set $C = T \setminus I$.

\begin{algorithm}
\caption{Transaction Partitioning}
\label{alg:partition}
\begin{algorithmic}[1]
\REQUIRE Transaction set $T = \{t_1, \ldots, t_n\}$, state snapshot $\sigma$
\ENSURE Independent set $I$, conflict set $C$
\STATE Compute $R(t_i), W(t_i)$ for each $t_i$ via static analysis
\STATE Build dependency graph $G = (T, E)$
\STATE $I \gets \text{GreedyMaxIndependentSet}(G)$
\STATE $C \gets T \setminus I$
\RETURN $I, C$
\end{algorithmic}
\end{algorithm}

\subsection{Parallel Execution}

Transactions in $I$ are dispatched to GPU streaming multiprocessors (SMs).
Each SM executes one transaction against a read-only snapshot of $\sigma_{\text{pre}}$ in GPU global memory.
State writes buffer in per-transaction write logs in SM-local shared memory.
No cross-transaction communication occurs during parallel execution.
The GPU kernel implements the full EVM instruction set in CUDA C++, including Cancun hard fork opcodes (EIP-4844).

\subsection{Conflict Detection and Re-execution}

After parallel execution, write logs are merged.
If two transactions in $I$ wrote to the same slot---a false negative in static analysis---a conflict is detected.

\begin{definition}[Execution Conflict]
A conflict exists between $t_i, t_j \in I$ if $W_{\text{actual}}(t_i) \cap W_{\text{actual}}(t_j) \neq \emptyset$.
\end{definition}

Conflicting transactions move to $C$ and the merge restarts.
False negatives occur in fewer than 0.3\% of transactions.

The conflict set $C$ executes \emph{sequentially} on CPU, ordered by block index, against the state from applying $I$'s merged writes to $\sigma_{\text{pre}}$.

\begin{theorem}[Deterministic Execution]
\label{thm:deterministic}
For any block $B$ with transaction set $T$, \cevm{} produces a unique state root $\sigma_{\text{post}}$ independent of partition $(I, C)$, provided $C$ executes sequentially in block order.
\end{theorem}

\begin{proof}
$I$ has pairwise-disjoint write sets; commutativity of disjoint updates ensures the merged state equals any sequential ordering of $I$.
$C$ then applies sequentially in canonical order.
Since sequential execution is deterministic and the base state is unique, $\sigma_{\text{post}}$ is unique.
\end{proof}

\section{GPU Execution Model}
\label{sec:gpu-model}

\subsection{Amdahl's Law for Blockchain Execution}

Let $f$ denote the sequential fraction (state conflicts + overhead) and $p$ the number of parallel execution units (GPU SMs).
By Amdahl's law:

\begin{equation}
S(p) = \frac{1}{f + \frac{1-f}{p}}
\label{eq:amdahl}
\end{equation}

\begin{definition}[Sequential Fraction]
\begin{equation}
f = \frac{|C|}{|T|} + \frac{\text{overhead}_{\text{analysis}} + \text{overhead}_{\text{merge}}}{W_{\text{total}}}
\end{equation}
\end{definition}

On Zoo L2, empirical measurements yield $f \in [0.03, 0.08]$, mean $\bar{f} = 0.047$.
For H100 with $p = 132$ SMs: $S(132) \approx 18.6\times$.
Theoretical bound: $S_{\max} = 1/f \approx 21.3\times$.

\subsection{Memory Bandwidth Bounds}

GPU EVM execution is bounded by memory bandwidth, not compute.
On H100's 3.35~TB/s HBM3:

\begin{equation}
\text{Throughput}_{\max} = \frac{B_{\text{mem}}}{b_{\text{avg}}} = \frac{3.35 \times 10^{12}}{1.46 \times 10^{3}} \approx 2.29 \times 10^{9} \text{ ops/sec}
\end{equation}

where $b_{\text{avg}} = 1.46$~KB is average memory traffic per gas operation.
This ceiling matches empirical measurements, confirming \cevm{} saturates GPU memory bandwidth.

\begin{theorem}[Linear Scaling]
\label{thm:linear}
For sequential fraction $f$ and $n$ independent transactions across $p$ SMs ($p \leq n$):
\begin{equation}
\Theta(p) = \Theta_{\text{base}} \cdot \frac{p}{1 + p \cdot f / (1 - f)}
\end{equation}
\end{theorem}

\section{Formal Verification}
\label{sec:formal}

All core properties are mechanically verified in Lean~4 (\texttt{GPU/EVMScaling.lean}).

\begin{theorem}[\texttt{linear\_scaling}]
\label{thm:formal-linear}
For independent set $I$ with $|I| = n$ and $p$ parallel units ($1 \leq p \leq n$):
\begin{equation}
\frac{\Theta(p)}{\Theta(1)} \geq \frac{p}{1 + (p-1) \cdot f}
\end{equation}
Proof by induction on $p$, showing each additional SM contributes positive marginal throughput for $f < 1$.
\end{theorem}

\begin{theorem}[\texttt{amdahl\_bound}]
\label{thm:formal-amdahl}
\begin{equation}
S(p) \leq \frac{1}{f} \quad \forall\, p \geq 1
\end{equation}
Proof constructs $\lim_{p \to \infty} S(p) = 1/f$ from Equation~\ref{eq:amdahl} and shows monotonicity.
\end{theorem}

\begin{theorem}[\texttt{deterministic\_reexecution}]
\label{thm:formal-deterministic}
For any two valid partitions $(I_1, C_1)$ and $(I_2, C_2)$ of $T$:
\begin{equation}
\sigma_{\text{post}}(I_1, C_1) = \sigma_{\text{post}}(I_2, C_2)
\end{equation}
Decomposed into three lemmas: \texttt{disjoint\_writes\_commute}, \texttt{sequential\_canonical}, and \texttt{merge\_equivalence}.
\end{theorem}

\section{Integration with Zoo L2}
\label{sec:integration}

\subsection{VM Architecture}

Zoo L2 runs three specialized VMs alongside \cevm{}:

\begin{enumerate}[leftmargin=*]
\item \textbf{DEX VM}: Order matching, AMM swaps, liquidity provision. Highest conflict rates due to shared pool state.
\item \textbf{FHE VM}: Homomorphic encryption on ciphertext. Near-zero conflicts (isolated encrypted state).
\item \textbf{AI Mining VM}: Proof-of-useful-computation. Independent result slots per miner.
\end{enumerate}

Specialized VMs are implemented as EVM precompiles (\texttt{0x0300}--\texttt{0x0320}) dispatched to dedicated GPU kernels.

Light validators execute only $C$ on GPU and verify $I$ via state-root comparison.

\subsection{Precompile Dispatch}

Precompile transactions are always placed in the conflict set to ensure execution against fully resolved base state:

\begin{algorithm}
\caption{Precompile-Aware Partitioning}
\label{alg:precompile}
\begin{algorithmic}[1]
\REQUIRE Transaction set $T$, precompile address set $P$
\STATE $T_{\text{pre}} \gets \{t \in T : \text{target}(t) \in P\}$
\STATE $I, C_{\text{evm}} \gets \text{Partition}(T \setminus T_{\text{pre}})$
\STATE $C \gets C_{\text{evm}} \cup T_{\text{pre}}$
\RETURN $I, C$
\end{algorithmic}
\end{algorithm}

\section{Related Work}
\label{sec:related}

The \Lux{} gpu-evm-whitepaper \cite{lux-gpu-evm} first proposed GPU acceleration for EVM execution on the \Lux{} C-Chain.
\cevm{} extends that work with precompile-aware partitioning, AI Mining scheduling, and formal verification.

\textbf{Monad} \cite{monad2024} implements optimistic parallel execution targeting CPU multi-core ($\sim$10$\times$ on 16 cores).
\textbf{Sei v2} \cite{sei2024} uses optimistic concurrency control for CPU-parallel EVM (5--12$\times$).
Both are limited by CPU core counts; \cevm{} achieves $23.8\times$ on a single GPU.

\textbf{MegaETH} \cite{megaeth2024} optimizes latency via specialized hardware sequencers with sequential execution, orthogonal to \cevm{}'s parallel throughput approach.

\textbf{Solana}'s Sealevel \cite{solana2020} requires upfront state-access declarations, incompatible with EVM semantics.
\textbf{Aptos} and \textbf{Sui} use Move-based resource ownership for compiler-verified parallelism, requiring new programming models.
\cevm{} preserves full EVM/Solidity compatibility.

The KEVM project \cite{kevm2018} provides complete EVM formal semantics in the K framework; \cevm{}'s Lean~4 proofs complement KEVM by covering the parallel execution model.

\section{Conclusion}
\label{sec:conclusion}

\cevm{} achieves 2.3~Gops/sec on a single GPU---$23.8\times$ over the fastest CPU EVM---while preserving full determinism through conflict detection and sequential re-execution.
Lean~4 verification provides mathematical guarantees that parallel execution produces identical state roots to sequential execution.
Multi-GPU linear scaling to 8.9~Gops/sec on four H100s meets the throughput demands of Zoo's AI-native L2 workloads.

Zoo Network's integration of \cevm{} with DEX VM, FHE VM, and AI Mining VM establishes a template for GPU-accelerated blockchain execution that maintains EVM compatibility while unlocking throughput previously exclusive to centralized systems.

Future work includes intra-precompile parallelism for DEX and FHE VMs, predictive conflict analysis via ML \cite{zoo-tokenomics}, ZK proof co-scheduling on GPU \cite{zoo-fhe}, and cross-L2 parallel execution across \Lux{} subnets \cite{zoo-dex}.
