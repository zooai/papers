\section{Integration with Zoo L2}

\label{sec:integration}

\subsection{VM Architecture}

Zoo L2 runs three specialized VMs alongside \cevm{}:

\begin{enumerate}[leftmargin=*]
\item \textbf{DEX VM}: Order matching, AMM swaps, liquidity provision. Highest conflict rates due to shared pool state.
\item \textbf{FHE VM}: Homomorphic encryption on ciphertext. Near-zero conflicts (isolated encrypted state).
\item \textbf{AI Mining VM}: Proof-of-useful-computation. Independent result slots per miner.
\end{enumerate}

Specialized VMs are implemented as EVM precompiles (\texttt{0x0300}--\texttt{0x0320}) dispatched to dedicated GPU kernels.

\subsection{Precompile Dispatch}

Precompile transactions are always placed in the conflict set to ensure execution against fully resolved base state:

\begin{algorithm}
\caption{Precompile-Aware Partitioning}
\label{alg:precompile}
\begin{algorithmic}[1]
\REQUIRE Transaction set $T$, precompile address set $P$
\STATE $T_{\text{pre}} \gets \{t \in T : \text{target}(t) \in P\}$
\STATE $I, C_{\text{evm}} \gets \text{Partition}(T \setminus T_{\text{pre}})$
\STATE $C \gets C_{\text{evm}} \cup T_{\text{pre}}$
\RETURN $I, C$
\end{algorithmic}
\end{algorithm}
